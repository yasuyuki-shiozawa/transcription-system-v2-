# 議会議事録管理システム

議会議事録の文字起こしデータを効率的に処理・管理するWebアプリケーションです。音声ファイルの自動文字起こし、話者管理、積算時間計算など、議事録作成の作業を大幅に効率化します。

## 主な機能

### 🎯 コア機能
- **音声ファイルアップロード**: MP3/WAV形式の音声を自動で文字起こし（OpenAI Whisper API使用）
- **セクション番号自動付与**: 話者ごとに連番でセクション番号を付与
- **複数データソース対応**: NOTTA・MANUSの文字起こしデータを統合管理
- **自動マッチング**: 同一内容のセクションを自動で紐付け
- **Word形式エクスポート**: 編集可能なWord文書として出力

### 🚀 新機能（v3.0）
- ドラッグ&ドロップによる音声ファイルアップロード
- リアルタイムプログレス表示
- エラーハンドリングの改善
- レスポンシブデザイン対応

### 🆕 新機能（v4.0）
- 👥 話者管理システム（議員・職員の事前登録）
- ⏱️ 話者別積算時間の自動計算
- 📝 Word書式のカスタマイズ機能
- 🔍 話者の自動認識（別名対応）

## クイックスタート

### 前提条件
- Node.js 18.0以上
- Docker Desktop（推奨）または PostgreSQL
- OpenAI APIキー（音声文字起こし機能を使用する場合）

### 30秒セットアップ（Docker使用）

```bash
# 1. リポジトリをクローン
git clone https://github.com/yasuyuki-shiozawa/transcription-system.git
cd transcription-system

# 2. 環境変数を設定
echo "OPENAI_API_KEY=your-api-key-here" > backend/.env

# 3. Dockerで起動
docker-compose up -d

# 4. ブラウザでアクセス
# http://localhost:3000
```

### ローカル開発環境

詳細なセットアップ手順は[開発者ガイド](docs/developer-guide/setup.md)を参照してください。

## 使い方

1. **セッションの作成**: 「新規セッション」ボタンをクリック
2. **データのアップロード**: 
   - 音声ファイル（MP3/WAV）をドラッグ&ドロップ
   - またはテキストファイル（NOTTA/MANUS形式）をアップロード
3. **データの確認・編集**: 自動処理されたセクションを確認
4. **エクスポート**: Word形式でダウンロード

詳細な使い方は[ユーザーガイド](docs/user-guide/usage.md)を参照してください。

### よくある質問

**Q: 音声ファイルの最大サイズは？**  
A: 100MBまでアップロード可能です。

**Q: 対応している音声形式は？**  
A: MP3およびWAV形式に対応しています。

**Q: 話者の識別はできますか？**  
A: 現在は話者識別機能は実装されていません。すべて「話者不明」として処理されます。
