# 議会議事録作成システム - セクション番号付与ツール

## 概要

このツールは、NOTTAで文字起こしされた議会議事録データに対して、話者の発言ごとにセクション番号を自動的に付与します。これにより、Manusでの処理時にセクションが飛ばされる問題を防ぎます。

## 機能

- NOTTAフォーマットの自動認識
- 話者変更の検出
- 連番でのセクション番号付与
- 統計情報の表示（話者数、発言回数など）

## インストール

```bash
# リポジトリのクローン
git clone [repository-url]
cd transcription-system

# 依存関係のインストール
npm install

# TypeScriptのビルド
npm run build
```

## 使い方

### 基本的な使用方法

```bash
# 入力ファイルを指定（出力ファイルは自動生成）
npm run process -- samples/sample_input.txt

# 入力と出力ファイルを指定
npm run process -- input.txt output.txt

# オプション付きで実行
npm run process -- input.txt output.txt --start 100 --digits 5
```

### コマンドラインオプション

- `--start, -s <number>`: セクション番号の開始値（デフォルト: 1）
- `--digits, -d <number>`: セクション番号の桁数（デフォルト: 4）
- `--help, -h`: ヘルプの表示
- `--version, -V`: バージョンの表示

## 入出力フォーマット

### 入力フォーマット（NOTTA）

```
話者 1 05:18

おはようございます。これより本日の会議を開きます。

話者 2 05:27

健次議員から遅刻の届け出が出ております。
```

### 出力フォーマット

```
【セクション：0001】[話者1][05:18]
おはようございます。これより本日の会議を開きます。

【セクション：0002】[話者2][05:27]
健次議員から遅刻の届け出が出ております。
```

## 注意事項

- 入力ファイルはUTF-8エンコーディングである必要があります
- NOTTAの出力フォーマット（`話者 X HH:MM`）に準拠している必要があります
- 話者番号は数字である必要があります

## トラブルシューティング

### よくある問題

1. **「入力ファイルがNOTTAの形式ではありません」エラー**
   - ファイルの最初の行が話者パターンになっているか確認してください
   - 文字エンコーディングがUTF-8であることを確認してください

2. **文字化け**
   - 入力ファイルの文字エンコーディングを確認してください
   - Windowsの場合、BOM付きUTF-8の可能性があります
