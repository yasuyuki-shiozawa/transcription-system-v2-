# よくある質問（FAQ）

## 一般的な質問

### Q: このシステムは何ができますか？
**A:** 議会議事録の作成を支援するシステムです。主な機能：
- 音声ファイル（MP3/WAV）の自動文字起こし
- NOTTA/MANUSの文字起こしデータの統合管理
- 話者ごとのセクション番号自動付与
- Word形式での議事録出力

### Q: 利用料金はかかりますか？
**A:** システム自体は無料で使用できます。ただし、音声文字起こし機能を使用する場合は、OpenAI APIの利用料金が別途かかります（約$0.006/分）。

### Q: どのような組織で使えますか？
**A:** 主に地方議会での利用を想定していますが、以下でも活用可能です：
- 各種委員会
- 審議会
- 理事会
- その他の会議体

## インストール・セットアップ

### Q: Docker Desktopは必須ですか？
**A:** いいえ、必須ではありません。以下の方法が選択できます：
1. Docker（推奨） - 最も簡単
2. ローカル環境 - Node.jsとPostgreSQLを個別にインストール
3. クラウド環境 - Render.comなどにデプロイ

### Q: Windows/Mac/Linuxで動作しますか？
**A:** はい、すべての主要OSで動作します：
- Windows 10/11（WSL2推奨）
- macOS 10.15以降
- Ubuntu 20.04以降

### Q: インストールにどれくらい時間がかかりますか？
**A:** 
- Docker使用時：約5分
- ローカル環境：約15-20分（依存関係のインストール含む）

## 音声文字起こし機能

### Q: OpenAI APIキーはどこで取得できますか？
**A:** 
1. [OpenAI Platform](https://platform.openai.com/)でアカウント作成
2. [API Keys](https://platform.openai.com/api-keys)ページで新規キー生成
3. 初回は$5分の無料クレジットが付与されます

### Q: 音声ファイルの最大サイズは？
**A:** 100MBまでアップロード可能です。それ以上の場合は、音声ファイルを分割してください。

### Q: 対応している音声形式は？
**A:** 現在対応している形式：
- MP3
- WAV
- （今後、M4A、WebMなども対応予定）

### Q: 話者の識別はできますか？
**A:** 現在は話者識別機能は実装されていません。すべての発言が「話者不明」として処理されます。話者情報は後から手動で編集してください。

### Q: 文字起こしの精度はどれくらいですか？
**A:** OpenAI Whisper APIを使用しており、日本語の認識精度は約95%以上です。ただし、以下の場合は精度が低下する可能性があります：
- 音質が悪い
- 複数人が同時に話している
- 専門用語が多い

## データ処理

### Q: NOTTA/MANUSとは何ですか？
**A:** 
- **NOTTA**: AI音声認識サービス。Web会議の文字起こしなどに使用
- **MANUS**: 議事録作成支援システム。多くの地方議会で採用

### Q: 既存のデータをインポートできますか？
**A:** はい、以下の形式に対応しています：
- NOTTAエクスポート形式（テキストファイル）
- MANUS出力形式（テキストファイル）
- 音声ファイル（MP3/WAV）

### Q: セクション番号とは何ですか？
**A:** 話者が変わるごとに付与される連番です。例：
```
【セクション：0001】[話者1][10:15]
議事を開始します。

【セクション：0002】[話者2][10:16]
本日の議題について説明します。
```

### Q: 自動マッチングとは？
**A:** NOTTAとMANUSの両方のデータがある場合、同じ内容のセクションを自動的に紐付ける機能です。これにより、複数ソースのデータを統合できます。

## 出力・エクスポート

### Q: Word以外の形式で出力できますか？
**A:** 現在はWord形式（.docx）のみ対応しています。今後、以下の形式も検討中です：
- PDF
- プレーンテキスト
- CSV

### Q: 出力されたWordファイルの編集は可能ですか？
**A:** はい、通常のWord文書として自由に編集できます。セクション番号は削除され、話者名とタイムスタンプのみが表示されます。

## トラブルシューティング

### Q: 「ポート3000が使用中」エラーが出ます
**A:** 他のアプリケーションがポートを使用しています。以下を試してください：
```bash
# Windowsの場合
netstat -ano | findstr :3000

# Mac/Linuxの場合
lsof -i :3000
```

### Q: データベース接続エラーが発生します
**A:** PostgreSQLが起動していない可能性があります：
```bash
# Dockerの場合
docker-compose ps
docker-compose restart postgres

# ローカルの場合
pg_ctl status
```

### Q: 音声アップロードが失敗します
**A:** 以下を確認してください：
1. OpenAI APIキーが正しく設定されているか
2. ファイルサイズが100MB以下か
3. ファイル形式がMP3/WAVか
4. インターネット接続が安定しているか

## セキュリティ・プライバシー

### Q: アップロードしたデータは安全ですか？
**A:** 
- ローカル環境：すべてのデータは自身のPCに保存されます
- 音声文字起こし：OpenAI APIに送信されますが、OpenAIは学習に使用しません
- 処理後の音声ファイルは自動的に削除されます

### Q: 機密性の高い会議で使用できますか？
**A:** ローカル環境で使用し、音声文字起こし機能を使わなければ、すべてのデータは組織内に留まります。機密性が特に高い場合は、オフライン環境での使用を推奨します。
