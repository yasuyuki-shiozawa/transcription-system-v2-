## ✅ 基本機能テスト

### 1. システム起動・アクセス

- [ ] システムが正常に起動する（http://localhost:3000）
- [ ] ホーム画面が正しく表示される
- [ ] ナビゲーションが機能する
- [ ] レスポンシブデザインが適切に動作する（PC、タブレット、スマートフォン）

**確認ポイント**: 
- ページの読み込み時間（5秒以内）
- デザインの崩れがないか
- エラーメッセージが表示されていないか

### 2. セッション管理

- [ ] 新しいセッションを作成できる
- [ ] セッション名を入力できる
- [ ] 開催日を設定できる
- [ ] セッション一覧が表示される
- [ ] 作成したセッションを開くことができる

**確認ポイント**:
- 日本語入力が正常に動作するか
- 必須項目の検証が機能するか
- セッション名に特殊文字が含まれても正常動作するか

- [ ] 複数ファイル同時ドロップでエラーが表示される
- [ ] エラーメッセージが分かりやすく表示される
- [ ] エラー解決方法が提示される

### パフォーマンス
- [ ] ページ読み込み: 5秒以内

- [ ] 編集操作の応答: 1秒以内
- [ ] ダウンロード処理: 5秒以内

### ユーザビリティ
- [ ] 操作が直感的で分かりやすい
- [ ] エラーメッセージが理解しやすい
- [ ] ヘルプやガイダンスが適切
- [ ] キーボード操作が可能

### アクセシビリティ
- [ ] スクリーンリーダーで読み上げ可能
- [ ] キーボードのみで操作可能
- [ ] 色のコントラストが適切
- [ ] フォントサイズが読みやすい

## 📊 テストケース

### テストケース1: 標準的な使用フロー
1. セッション作成
2. NOTTAファイルアップロード（500KB、.txt）
3. Manusファイルアップロード（300KB、.docx）
4. データ比較画面で内容確認
5. 3つのセクションを除外
6. Word形式でダウンロード

**期待結果**: 全工程が5分以内で完了、出力ファイルが正常

### テストケース2: エラーハンドリング
1. 非対応ファイル（.pdf）をアップロード
2. 大きすぎるファイル（15MB）をアップロード
3. 空ファイルをアップロード
4. ネットワーク切断状態での操作

**期待結果**: 各エラーで適切なメッセージ表示、システムがクラッシュしない

### テストケース3: 大量データ処理
1. 大容量ファイル（8MB）をアップロード
2. 100セクション以上のデータで編集
3. 50セクション除外後のダウンロード

**期待結果**: 処理が正常完了、パフォーマンス低下が許容範囲内
