# 議会議事録作成システム - 本番環境セットアップガイド

## 概要
このガイドは、議会議事録作成システムを社内の他の事務員と共有するための設定手順を説明します。

## アクセス情報

### サーバーPC（ホスト）
- **開発環境**: http://localhost:3000 （テスト用）
- **本番環境**: http://localhost:3003 （共有用）

### 他のPC（クライアント）から
- **本番環境**: http://[サーバーPCのIPアドレス]:3003
  - 例: http://192.168.1.100:3003

## 起動手順

### 1. 本番環境の起動（サーバーPC）

#### 方法A: バッチファイルを使用（推奨）
1. エクスプローラーで `transcription-system\scripts\` フォルダを開く
2. `start-production.bat` を右クリック
3. 「管理者として実行」を選択

#### 方法B: コマンドラインから
```bash
cd /mnt/c/Users/shioz/Downloads/transcription-system
./scripts/start-production.sh
```

### 2. IPアドレスの確認
起動スクリプトが自動的にIPアドレスを表示します。
手動で確認する場合：

#### Windows（PowerShell）
```powershell
ipconfig | Select-String "IPv4"
```

#### WSL2
```bash
hostname -I
```

### 3. ファイアウォールの設定

初回のみ、Windows Defenderファイアウォールでポート3003を開放する必要があります：

1. Windows Defenderファイアウォールを開く
2. 「詳細設定」をクリック
3. 「受信の規則」→「新しい規則」
4. 以下を設定：
   - 規則の種類: ポート
   - プロトコル: TCP
   - 特定のローカルポート: 3003
   - 接続を許可する
   - プロファイル: プライベート（社内ネットワークのみ）
   - 名前: TranscriptionSystem-Production

## 使用方法（他のPCから）

### アクセス手順
1. ブラウザ（Chrome推奨）を開く
2. アドレスバーに `http://[サーバーPCのIP]:3003` を入力
   - 例: `http://192.168.1.100:3003`
3. ブックマークに登録しておくと便利です

### 推奨ブラウザ
- Google Chrome（推奨）
- Microsoft Edge
- Firefox

## トラブルシューティング

### アクセスできない場合

1. **ping確認**
   ```cmd
   ping [サーバーPCのIP]
   ```

2. **ファイアウォール確認**
   - Windows Defenderファイアウォールでポート3003が開放されているか確認
   - ウイルス対策ソフトがブロックしていないか確認

3. **サーバー状態確認**
   - サーバーPCでブラウザから http://localhost:3003 にアクセスできるか確認

### パフォーマンスが遅い場合

1. **VPN最適化の確認**
   - 本番環境は自動的にVPN最適化モードで起動します

2. **ネットワーク速度**
   - 有線LAN接続を推奨
   - Wi-Fiの場合は5GHz帯を使用

## セキュリティ注意事項

1. **社内ネットワーク限定**
   - このシステムは社内ネットワークでのみ使用してください
   - VPN経由でのアクセスも可能です

2. **データの取り扱い**
   - 議事録データは機密情報として扱ってください
   - 定期的にバックアップを取ることを推奨します

## サーバーの停止

### バッチファイルを使用
1. `scripts\stop-production.bat` を実行

### 手動停止
```bash
./scripts/stop-production.sh
```

## 自動起動設定（オプション）

PCを再起動しても自動的にサーバーが起動するようにする場合：

1. タスクスケジューラを開く（`taskschd.msc`）
2. 「タスクの作成」
3. 以下を設定：
   - 名前: TranscriptionSystem-Production
   - トリガー: システム起動時
   - 操作: `C:\Users\shioz\Downloads\transcription-system\scripts\start-production.bat`
   - 「最上位の特権で実行する」にチェック

## サポート

1. エラーメッセージのスクリーンショット
