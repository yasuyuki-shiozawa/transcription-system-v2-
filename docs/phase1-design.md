# 議会議事録作成システム 設計書

## 1. プロジェクト概要

### 1.1 背景
- 議会議事録の作成業務を受注し、継続的に反訳業務を行う
- nottaによる文字起こしデータとManusが作成した議事録を効率的に処理する必要がある
- 現在、Manusがセクションを飛ばしてしまう問題が発生している

### 1.2 目的
発言ごとにセクション番号を付与し、Manusが確実にすべてのセクションを認識・処理できるようにする

### 1.3 開発フェーズ
**現在は第1フェーズ（①）に取り組む**

## 2. システム全体構成（最終形）

1. ①nottaの文字起こしデータにセクション番号を入れる **← 今回実装**
2. ②Manusのデータにセクション番号が反映される（Manus側で対応）
3. ③①及び②をシステムに取り込む
4. ④録音データを聞きながら両データを一画面で参照
5. ⑤両データを見ながら修正
6. ⑥必要なセクションの選択
7. ⑦タイムスタンプの手入力
8. ⑧整形作業

## 3. 第1フェーズ（①）の詳細設計

### 3.1 機能要件
nottaの文字起こしデータに対して、話者の変更ごとにセクション番号を自動付与する

### 3.2 入力仕様
- **形式**: テキストファイル（.txt）またはWordファイル（.docx）
- **内容**: nottaが出力した文字起こしデータ
- **構造**: 
  ```
  話者 X HH:MM
  発言内容
  ```

### 3.3 出力仕様
- **形式**: テキストファイル（.txt）
- **構造**:
  ```
  【セクション：XXXX】[話者X][HH:MM]
  発言内容
  ```
- **セクション番号**: 4桁の連番（0001から開始）

### 3.4 処理フロー
1. ファイルの読み込み
2. 話者パターンの検出（正規表現: `話者 \d+ \d{2}:\d{2}`）
3. 各発言にセクション番号を付与
4. フォーマットされたテキストの出力

### 3.5 技術仕様
- **言語**: TypeScript/JavaScript（Node.js環境）
- **主要ライブラリ**:
  - ファイル処理: fs/promises
  - Word文書処理: mammoth（必要な場合）
- **文字コード**: UTF-8

### 3.7 エラーハンドリング
- ファイル読み込みエラー
- 不正なフォーマットの検出
- 文字エンコーディングエラー

### 3.8 テスト要件
- 単体テスト：各関数の動作確認
- 統合テスト：実際のnottaデータでの動作確認
- エッジケース：話者番号が連続しない場合、時刻形式が異なる場合

2. **拡張性**

## 6. サンプル入出力

```

おはようございます。これより本日の会議を開きます。ただいまの出席議員数28名でございます。毎週、

話者 2 05:27

健次議員から遅刻の届け出が出ております。

話者 3 05:32

本日は議事日程第5号によって議事を、
```

```
【セクション：0001】[話者1][05:18]
おはようございます。これより本日の会議を開きます。ただいまの出席議員数28名でございます。毎週、

本日は議事日程第5号によって議事を、
```
