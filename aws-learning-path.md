# AWS学習ロードマップ（議事録システムを題材に）

## なぜ議事録システムから始めないほうが良いか

1. **本番システムで学ぶリスク**
   - 設定ミスでサービス停止
   - 誤った設定で高額請求
   - セキュリティホール

2. **もっと簡単な選択肢がある**
   - Vercel（自動スケーリング）
   - Railway（簡単デプロイ）
   - Render（Docker対応）

## AWSを学ぶための段階的アプローチ

### Phase 1: 基礎学習（1-2週間）
- **AWS無料利用枠**で練習
- 簡単な静的サイトから始める
- 月額予算アラートを必ず設定

### Phase 2: 開発環境構築（2-4週間）
```
1. EC2でWebサーバー構築
2. RDSでデータベース作成
3. S3で画像保存
4. CloudFrontでCDN設定
```

### Phase 3: 議事録システム移行（1ヶ月後）
- 開発環境で十分テスト
- 段階的に移行
- 旧システムと並行運用

## 学習リソース

### 無料で学べる
- [AWS公式トレーニング](https://aws.amazon.com/jp/training/)
- [AWS Hands-on for Beginners](https://aws.amazon.com/jp/aws-jp-introduction/)
- YouTube: AWS公式チャンネル

### 資格取得を目指す
- AWS Cloud Practitioner（基礎）
- AWS Solutions Architect Associate（実務）

## コスト管理のコツ

### 必須設定
```bash
# 1. 予算アラート設定
AWS Budgets で月1,000円でアラート

# 2. 無料利用枠アラート
Billing で無料枠超過を監視

# 3. リソースの自動停止
Lambda で夜間/週末の自動停止
```

### よくある失敗と対策
| 失敗例 | 請求額 | 対策 |
|--------|--------|------|
| EC2停止忘れ | 5,000円/月 | 自動停止設定 |
| EIP放置 | 3,000円/月 | 未使用は即削除 |
| NAT Gateway | 8,000円/月 | 開発では使わない |
| RDS Multi-AZ | 10,000円/月 | 本番のみ使用 |

## 結論：おすすめの進め方

### 今すぐ議事録システムを使いたい場合
```
1. Vercel + Supabase でまず運用開始（0円）
2. 並行してAWSを学習
3. 十分理解してから移行を検討
```

### AWSスキルを身につけたい場合
```
1. 議事録システムは簡単な方法で運用
2. 別の練習プロジェクトでAWS学習
3. スキルが身についてから統合
```

### 予算に余裕がある場合
```
1. AWS認定パートナーに構築依頼
2. 運用しながら学習
3. 徐々に自社管理に移行
```
