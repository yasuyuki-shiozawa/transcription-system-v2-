# エラーメッセージコンポーネント 設計仕様

**作成者**: Aphrodite（UI/UXデザイナー）  
**日付**: 2025-06-26  
**バージョン**: 1.0

## 概要

現在のシステムのエラーハンドリングを改善し、ユーザーフレンドリーなエラー体験を実現するためのコンポーネント設計仕様書です。

## 設計原則

### 1. 明確性（Clarity）
- エラーの原因を具体的に説明
- 解決方法を提示
- 専門用語を避け、わかりやすい言葉を使用

### 2. 親しみやすさ（Friendliness）
- 責めるような表現を避ける
- 建設的なトーンで案内
- ユーザーをサポートする姿勢

### 3. 行動可能性（Actionability）
- 次に何をすべきかを明示
- 具体的な手順を提供
- 代替手段の提案

## コンポーネント仕様

### エラーレベル

#### 🔴 Critical（重大）
- システムエラー、致命的な問題
- 赤色（`--color-error-500`）
- アイコン: 警告マーク

#### 🟡 Warning（警告）
- ユーザーの注意が必要
- 黄色（`--color-warning-500`）
- アイコン: 注意マーク

#### 🔵 Info（情報）
- 補足情報、軽微な問題
- 青色（`--color-info-500`）
- アイコン: 情報マーク

### コンポーネント構造

```typescript
interface ErrorMessageProps {
  level: 'critical' | 'warning' | 'info';
  title: string;
  message: string;
  action?: {
    label: string;
    onClick: () => void;
  };
  onClose?: () => void;
  persistent?: boolean;
}
```

## 具体的なエラーメッセージ案

### 1. ファイルアップロードエラー

#### 現在（Before）
```
「アップロードに失敗しました」
```

#### 改善案（After）
```
🔴 ファイルのアップロードができませんでした

ファイルサイズが100MBを超えているか、対応していない形式の可能性があります。

📝 解決方法:
1. ファイルサイズを確認してください（100MB以下）
2. MP3またはWAV形式のみ対応しています
3. インターネット接続を確認してください

[もう一度試す] [ファイルを変更]
```

### 2. ネットワークエラー

#### 改善案
```
🟡 サーバーに接続できません

インターネット接続に問題があるか、サーバーが一時的に利用できない状態です。

📝 解決方法:
1. インターネット接続を確認してください
2. しばらく待ってから再試行してください
3. 問題が続く場合は管理者にお問い合わせください

[再試行] [オフラインで続行]
```

### 3. バリデーションエラー

#### 改善案
```
🔵 入力内容をご確認ください

セッション名は必須項目です。わかりやすい名前を入力してください。

📝 入力例:
- 「令和6年第1回定例会」
- 「2025年3月委員会」

[セッション名を入力]
```

## インタラクション仕様

### アニメーション
- **表示**: Fade in + Slide down（300ms）
- **非表示**: Fade out（200ms）
- **ホバー**: 軽微な影の変化

### 自動非表示
- **Info**: 5秒後に自動非表示
- **Warning**: 8秒後に自動非表示
- **Critical**: 手動クローズのみ

### キーボード操作
- `Escape`: エラーメッセージを閉じる
- `Enter`: 主要アクションボタンを実行
- `Tab`: フォーカス移動
