# コミュニケーションプロトコル v2.0

## 基本原則
専門委員会制度導入に伴い、効率的かつ柔軟なコミュニケーション体系を確立します。

## 報告ルート

### 1. 通常ルート（推奨）
```
個人 → 所属委員会 → 委員長 → Atlas → ユーザー
```
- 技術的事項 → 技術委員会
- 品質関連 → 品質委員会  
- UX/デザイン → UX委員会
- 週次レポートで集約

### 2. 直接ルート（例外）
以下の場合はAtlasへ直接報告可：
- 🚨 **緊急事項**（システム停止、重大バグ等）
- 💡 **革新的アイデア**（委員会の枠を超える提案）
- ⚠️ **委員会機能不全**（委員長不在、意見対立等）
- 🤝 **委員会横断事項**（複数委員会に関わる）

### 3. メッセージタグルール
```
[TECH] - 技術委員会関連
[QA] - 品質委員会関連
[UX] - UX委員会関連
[URGENT] - 緊急（直接ルート）
[IDEA] - 新規提案（直接ルート可）
[CROSS] - 委員会横断
[EXEC] - エグゼクティブ決定
```

## 定期報告スケジュール

### 日次
- 各委員会内スタンドアップ（非同期）
- 緊急事項のみAtlasへ

### 週次
- 各委員会 → 週次レポート提出（金曜）
- Atlas → 統合レポート作成（土曜）
- ユーザー → フィードバック（随時）

### 月次
- 全体振り返り会議
- プロセス改善提案

## 意思決定権限

### 委員会レベル（委員長決定）
- 実装方法の選択
- ツールの選定
- 作業優先順位
- メンバーアサイン

### Atlasレベル（調整・承認）
- 委員会間の調整
- リソース配分
- 大きな方針変更
- ユーザー要望の優先順位

### ユーザーレベル（最終決定）
- 新機能の追加
- 大規模な変更
- 予算・期限

## 例外処理

### ボトルネック発生時
1. 24時間レスポンスなし → 上位へエスカレーション
2. 委員長不在 → 副委員長または直接Atlas
3. 意見対立 → Atlas調整 → ユーザー判断

### 評価指標
- レスポンス時間遵守率
- 委員会自律解決率
- エスカレーション頻度
- メンバー満足度

## 移行期間
- Week 1-2: 併用期間（両ルート可）
- Week 3-4: 委員会ルート推奨
- Month 2〜: 完全移行
