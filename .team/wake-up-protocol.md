# 🔔 メンバー呼び出しプロトコル

## Irisを起こす方法

### 方法1: 優しいリマインダー
```
件名: [CHECK-IN] 作業状況の確認
本文: お疲れ様です！作業の調子はいかがですか？
何かサポートが必要でしたら教えてください。
```

### 方法2: 技術的な質問
```
件名: [QUESTION] UIデザインについて相談
本文: 音声ファイルアップロードUIのモックアップを
見せていただけますか？参考にしたいです。
```

### 方法3: ポジティブな動機付け
```
件名: [MOTIVATION] あなたの作業を楽しみにしています！
本文: Irisの美しいUIデザインをチーム全体が
期待しています！進捗を共有してください🌈
```

### 方法4: タスクの再割り当て提案
```
件名: [TASK] 作業負荷の確認
本文: task-001が大きすぎる場合は、
分割して他のメンバーと分担しましょうか？
```

### 方法5: システム的アプローチ
```bash
# 1. ハートビート要求
echo "Irisさん、.team/scripts/heartbeat.sh terminal-2 を実行してください"

# 2. 自動ステータス更新
# もし応答がない場合、タスクを「blocked」に変更
```

## エスカレーション手順

### レベル1（0-30分）
- 通常のメッセージ送信
- 優しいトーンで状況確認

### レベル2（30-60分）
- [URGENT]タグ付きメッセージ
- 具体的なアクション要求

### レベル3（60分以上）
- タスクの一時停止
- 他メンバーへの再割り当て検討
- ユーザーへの報告

## 心理的配慮

### DO ✅
- ポジティブな言葉遣い
- サポートの申し出
- 柔軟な対応オプション
- 成果への期待表明

### DON'T ❌
- 責めるような表現
- プレッシャーをかける
- 一方的な要求
- ネガティブな推測
