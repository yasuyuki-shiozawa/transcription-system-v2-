## 技術的課題の整理

### 🔴 緊急対応が必要な課題

#### 1. フロントエンド-バックエンドAPI連携の最終確認
**課題詳細**:
- API仕様の最終確認
- エラーレスポンス形式の統一
- 進捗通知方法の決定

**技術仕様**:
```
エンドポイント: POST /api/sessions/{id}/upload/audio/{source}
パラメータ: source = 'NOTTA' | 'MANUS'
リクエスト: multipart/form-data
ファイルサイズ: 最大100MB
対応形式: .mp3, .wav
```

**Irisへの確認事項**:
- フロントエンドからのAPI呼び出し実装状況
- エラーハンドリングの統一方法
- 進捗表示の実装方法

#### 2. Whisper APIのエラーハンドリング統一
**現状実装**:
- 基本的なtry-catch実装済み
- transcriptionService.tsにエラー処理あり

**改善が必要な点**:
- API制限エラー対応
- 音声ファイル形式エラー処理  
- ネットワークエラー時のリトライ機構

**技術的解決策案**:
```typescript
const retryConfig = {
  maxRetries: 3,
  retryDelay: 1000,
  backoffMultiplier: 2
};
```

#### 3. 本番環境でのパフォーマンス最適化
**Hermesへの確認事項**:

- ログ監視設計
- パフォーマンステスト戦略
- 並行処理数の上限設定

## 委員会の決定事項

### 技術標準
1. **ファイルサイズ上限**: 100MB
2. **対応音声形式**: .mp3, .wav
3. **API タイムアウト**: 120秒
4. **データベース**: 既存SQLite + Prisma継続

### 開発プロセス
1. **コードレビュー**: 必須（詳細は後日策定）
2. **テスト**: CI/CDパイプライン活用
3. **ドキュメント**: API仕様書の継続メンテナンス
